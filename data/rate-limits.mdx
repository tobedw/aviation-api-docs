## Plans

| Plan | Requests/month | Max page size | Bulk download |
|---|---|---|---|
| Community | 20,000 | 50 | Filter required |
| Startup | 200,000 | 100 | ✅ |
| Pro | 1,000,000 | 500 | ✅ |
| Enterprise | Unlimited | 500 | ✅ |

**Bulk download** means querying an endpoint without any filter. On the Community plan, at least one filter is required for `/v1/airlines`, `/v1/airports`, and `/v1/cities`.

## Rate limit headers

Every response includes:

| Header | Description |
|---|---|
| `X-RateLimit-Limit` | Total requests allowed this month |
| `X-RateLimit-Remaining` | Requests remaining this month |
| `X-RateLimit-Reset` | Reset date (`YYYY-MM-DD`, always the 1st of next month) |

```bash
HTTP/1.1 200 OK
X-RateLimit-Limit: 20000
X-RateLimit-Remaining: 19873
X-RateLimit-Reset: 2026-05-01
```

Limits reset automatically on the 1st of each month.

## Pagination

Use `limit` and `offset` to paginate results:

```bash
# Page 1
GET /v1/airlines?country_code=DE&limit=50&offset=0

# Page 2
GET /v1/airlines?country_code=DE&limit=50&offset=50
```

| Parameter | Default | Description |
|---|---|---|
| `limit` | `50` | Results per page (capped by plan) |
| `offset` | `0` | Number of results to skip |

The response always includes `total`, `page`, `limit`, and `truncated` fields so you know when to fetch the next page.

## Error responses

| Status | Meaning |
|---|---|
| `400` | Missing required filter |
| `404` | Unknown endpoint |
| `405` | Method not allowed (only GET) |
| `500` | Internal server error |
